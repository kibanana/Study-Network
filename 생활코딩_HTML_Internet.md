### HTML이 중요한 이유
기초는 응용으로 가기 위한 길목이기도 하지만, 기초 그 자체만으로 할 수 있는 것도 많다.

웹의 핵심적인 철학은 **접근성(accessibility)** 이다. 웹은 모든 운영체제에서 동작하고, 웹의 소스코드는 누구나 볼 수 있어야 한다. 또한 저작권이 없다.

하지만 시각장애가 있는 사람은 시각화되어 있는 정보를 청각화 하여 정보를 접하게 되는데, 스크린 리더 프로그램이나 각종 보조장치를 이용한다. 

만약 예쁜 글자를 쓰고 싶어서 글자를 이미지화해서 페이지에 집어넣는다면, 시각장애가 있는 사람에게 그 정보는 없는 정보가 된다.


## Internet

내가 만든 웹페이지를 인터넷을 통해 누구나 가져갈 수 있도록 해야 한다.

### 웹의 역사 -> Primitive Web

가장 중요한 정보기술로 군림하여 복잡해졌다.

그래서 웹을 처음 접하면 굉장히 막막할 수 있다.

ex) 우리가 웹의 역사를 거슬러 올라가서 웹이 처음 세상에 나왔을 적에 도착한다면 그곳에 웹의 본질이 있지 않을까?

-> **웹이 웹이기 위해 필요한, 필수적인 것은 있으며 군더더기는 없는 상태** -> **Primitive Web**

우리는 이 수업을 통해 정보를 보는 관점을 넓힐 것이다. 우리가 단순히 정보의 소비자로 산다면 이런 행위는 필요 없을 것이다. 하지만, 우리는 언젠가 이런 혁명적인 무엇인가를 직접 만들수도 있다. 그러려면 이 혁명이 어떻게 시작(Birth), 지속(Time goes by), 소멸(Death)되는지 자세히 알아야 한다.

비유적으로 차이점을 말하자면, 인터넷이 도시라면 웹은 그 도시 위에 있는 건물 하나다.

인터넷이 운영체제라면 웹은 그 운영체제 위에서 돌아가는 응용프로그램 하나다.

- 1960년 : 인터넷 등장

  핵이 화두였다. 당시 인류는 핵전쟁을 치른 직후였다. 미국은 핵전쟁에 대비할 시뮬레이션을 돌려봤지만, 통신 분야에서 심각한 약점이 다수 발견되었다. 당시의 통신 시스템은 중앙집중식이어서 핵공격을 당하면 통신이 마비될 위험이 있었다. 따라서 핵공격에도 괜찮을 강력한 통신 시스템을 개발했다.
  
  그것이 인터넷이다. 중앙 없이 각 노드(통신장치)가 연결되는 형태다. 수많은 통신장치가 분산되어서 전화국 역할을 하기 때문에 한 통신장치가 망가져도 나머지 통신은 정상적으로 이루어지는 것이다.
  
  수많은 서비스(ex. Google, Facebook)를 묵묵히 지탱하는 인프라 역할을 한다.
  
  이때까지 사람들은 인터넷의 존재에 대해 잘 몰랐다. 인터넷은 연구소, 대학, 군대 같은 거대한 기관에서 사용하는 통신시스템이었기 때문이다.

- 1990년 : 웹 등장

  웹이 출현하면서 인터넷은 급속도로 대중화된다.
  
  웹의 고향은 스위스다. 왜일까? 스위스의 제네바에 유럽 입자 물리 연구소(CERN)라는 아주 저명한 물리학 연구소가 있고, 이곳에는 강입자 가속기라는 장치가 있다. 이 장치는 인류가 지금껏 만든 장치 중에 가장 거대하고, 비싸고, 복잡한 장치 중 하나다.
  
  이 장치로는 아주 작은 입자를 본다. 둘레가 27km다.
  
  쇠로 된 튜브 안에서 작은 입자를 고속으로 회전시켜서 입자와 입자가 충돌했을 때 어마어마한 에너지가 방출되면서 새로운 작은 입자를 발견하게 될 것이라는 가설을 세웠다. 그 입자가 바로 힉스 보선이다.
  
  힉스라는 이름의 박사가 이러이러한 것이 존재할 것이라는 가설을 세웠다. 하지만 당시에는 그것을 확인할 수 있는 방법이 없었기 때문에 계속 가설로 있었다.
  
  그리고 최근 몇 년 전에 힉스가 존재한다는 발표가 났다. 
  
  수십년전 전에는 정보라는 단어조차 익숙하지 않을 정도로 정보 기술이 미숙한 시절이었는데, 그 당시 유럽 입자 물리 연구소에는 수많은 엔지니어와 연구원들이 모여서 현재와 비슷하게, 정보의 바다에서 허우적대고 있었다.
 
  1980년부터 '팀 버너스 리'라는 프로그래머가 비정규직으로 취직하게 되고, 처음으로 Enquire라는 프로그램을 만든다. 이 프로그램은 누군가에게 넘겨졌지만 지금은 사라져서 존재하지 않는다. 하지만 이 프로그램이 웹의 전신 중 하나가 되었다.
  
  그렇게 팀 버너스 리는 10년동안 연구소를 나왔다 들어갔다를 반복하면서 웹의 전신이 되는 프로그램을 만드는 것을 반복한다. 그리고 1990년, 연구소에 인터넷이 들어왔다. 팀 버너스 리는 이전부터 구상하던 것에 인터넷을 더해서 웹을 만들기 시작한다.
  
  1990년 10월에 최초로 웹페이지를 만드는 편집기를 만든다. 1990년 11월에 세계 최초의 웹 브라우저인 World Wide Web이라는 프로그램을 만든다.

### 서버와 클라이언트
  
인터넷이 동작하기 위해서는 컴퓨터가 최소 2대 이상 있어야 한다.

팀 버너스 리는 두 대의 컴퓨터를 장만하고, 두 개의 프로그램을 개발한다.

-> Web Browser, Web Server(`http://info.cern.ch`) 

웹서버가 설치되어 있는 컴퓨터의 하드디스크에 어느 디렉터리에 index.html 이라는 파일이 존재한다.

웹브라우저에서 `http://info.cern.ch/index.html`을 입력하고 엔터를 누르면, 

1. 인터넷을 통해 전기적인 신호를 Web Server가 설치된 컴퓨터에 보낸다. 그 신호 안에는 Web Browser가 원하는 정보에 대한 설명이 담겨있다.
2. Web Server는 하드디스크에서 index.html 이라는 파일을 찾아서 전기적 신호로 바꾸고, Web Browser가 설치된 컴퓨터로 보낸다.
3. Web Browser가 설치된 컴퓨터에 index.html 파일의 내용이 도착한다.
4. Web Browser가 도착한 내용을 읽어서 화면에 표시한다.

Web Browser가 설치된 컴퓨터는 request(요청), Web Server가 설치된 컴퓨터는 response(응답) 한다.

각각 Client, Server다.


### Web Hosting -> Web Server를 설치하고 서비스를 올리는 작업을 대행

직접 컴퓨터에 웹서버를 설치하고 인터넷과 연결해서 외부에서 웹 서버에 올린 서비스에 접근하기 위해서는 여러 작업을 해야 한다. 쉽지 않다. 그래서 이런 일을 대신해주는 회사들이 있다.

**인터넷이 연결되어 있는 컴퓨터 = 웹서버를 운영하기 위한 컴퓨터**를 호스트라고 한다. 이것을 빌려주는 업체를 웹 호스팅 업체라고 한다. 

이 예시가 Github다. Github는 소스코드 저장소이지만 동시에 웹서버로 쓰도록 설정할 수도 있다(`github.io`).


### Web Server 운영

> 만약 문제를 해결하는 데에 몇날 며칠이 걸려서 결국 문제를 해결했다면, 우리는 단지 그 문제만 해결한 것이 아니라 그 해결 과정에서 수많은 지식과 경험을 보답으로 얻게 된다.

### 웹서버의 요모조모
주소창에 `http://127.0.0.1/index.html` 입력 후 엔터

index.html 파일은 어디에 있을까? Apache가 설치된 디렉터리 내에서 index.html 파일을 볼 수 있다.

이 index.html 파일의 HTML 코드를 수정하면 수정한 대로 브라우저에서 볼 수 있는 페이지에 적용된다.

그리고 이 index.html 파일에는 `http://127.0.0.1/index.html`처럼 접근할 수도, `C: ~~~ /index.html`로 접근할 수도 있는데, 이 두 개의 차이점은 무엇일까? 정말 굉장한 차이가 있다.

`http`는 웹서버와 웹브라우저가 서로 정보를 주고받기위한 약속이다. -> Hyper Text Transfer Protocol

서로 다른 컴퓨터에 있는 웹서버와 웹브라우저가 서로 통신하기 위해서는 반드시 **http**를 사용해야 한다.

외부에서 내부의 데이터에 접근하기 위해서는 IP Address를 알아야 한다.
