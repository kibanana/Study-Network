# 네트워크 (Network)

- Net(그물) + Work(일하다) : 컴퓨터 간의 대화 의미
- 시설 상호간에 형성되는 조직

**네트워크**는 **송신자의 메시지를 수신자에게 전달하는 과정**으로, 
**한 지점에서 원하는 다른 지점으로 의미 있는 정보를 *보다 빠르고 정확하고 상대방이 이해할 수 있도록* 전송하는 것**을 의미한다.

또한 메시지를 전송하거나 메시지를 받을 수 있는 것을 의미하며, 사용자들이 스마트폰이나 데스크톱 컴퓨터를 사용하여 
인터넷을 사용할 수 있는 것은 네트워크가 있기 때문에 가능하다.

네트워크는 유선의 케이블을 컴퓨터에 연결해서 사용할 수도 있고, 케이블을 사용하지 않고 무선으로도 사용할 수 있다. 
즉, **연결 형태에 따라 유선 네트워크와 무선 네트워크로 구분된다.**

### 네트워크 거리에 따른 구분

네트워크를 분류할 때, 신호가 전송되는 거리에 따라서 네트워크를 분류한다.

![image](https://user-images.githubusercontent.com/37951612/79365876-359c4380-7f86-11ea-80de-3aacc20596d5.png)


- **PAN(Personal Area Network)**

  3m ~ 5m 이내, 인접지역 간의 통신 방법으로 거리가 짧다는 특성이 있다. 
  짧은 거리로 인해 보통 유선보다 무선 WRAN(ex. 블루투스)이 많이 활용된다.

- **LAN(Local Area Network)**

  50m 이내, 근거리 영역의 네트워크로 동일한 지역(ex. 기업이나 캠퍼스)내의 고속의 전용 회선으로 연결하여 구성하는 통신망이다.
  
  Client & Server와 P2P(Peer to Peer) 모델이며, WAN보다 빠른 통신 속도를 지닌다.

- **MAN(Metropolitan Area Network)**

  LAN과 WAN의 중간 형태의 네트워크다. 데이터, 음성, 영상 등을 지원하기 위해 개발된 네트워크다.
  
  20~50km 이내(최대 75km), 서울시 행정망 등에서 사용한다.
  
  전송매체는 동축 케이블, 광 케이블이며 DQDB(Distributed Queue Dual Bus, MAN 네트워크로 개발된 프로토콜)를 사용한다.

- **WAN(Wide Area Network)**

  1개 국가 이내, 광대역 네트워크망으로 서로 관련이 있는 LAN들간의 상호 연결망이다. LAN에 비해 선로 에러율이 높고, 전송 지연이 크다  .
  
  WAN 설계 시 전송 효율과 특성을 고려해야 한다. 두 목적지 간을 최단 경로로 연결시켜주는 라우팅 알고리즘이 중요하다. 
  제한된 트래픽 조건 하에서 흐름을 제어하고, 과도한 지연을 제거한다.
  
  KT의 서비스망 등에서 사용한다. 

- **GAN(Global Area Network)**

  세계적인 망, Internet에서 사용한다.
  
### 데이터 전송(Data Transfer) 방식
데이터 전송 방식이란 **데이터를 어떠한 방식으로 보내느냐**는 것이다.  
ex) 무전기 : 송신자와 수신자가 음성을 전송한다. 하지만 한 사람이 말하는 도중 한 사람은 말할 수 없고 듣기만 해야한다. 
이렇게 동시에 데이터를 주고받을 수 없는 전송 방식을 반이중 방식이라고 한다.

- **단방향(Simplex) 통신**

  데이터 전송만 가능, 수신은 불가능하다.

- **반이중(Half Duplex) 통신** ex) 무전기

  데이터를 전송, 수신할 수 있지만 동시에 할 수는 없다.

- **전이중(Full Duplex) 통신** ex) 전화기

  동시에 데이터를 송신 및 수신할 수 있다.


### 네트워크(통신망) 구조에 따른 구분 -> 네트워크 토폴로지(Topology)
컴퓨터의 **네트워크 요소들을 물리적으로 연결해놓은 것, 또는 그 연결 방식**을 말한다.

정보 통신망의 구성이라는 것은 데이터 통신을 위해 각각의 정보 단말장치(컴퓨터)를 어떤 형태로 연결할 것인가에 대한 내용이다.

![image](https://user-images.githubusercontent.com/37951612/79114782-21f2b080-7dbf-11ea-8fd6-c226f0b190b7.png)

- **Tree Topology(계층형 토폴로지)**

  트리 형태로 정보 통신망을 구성하는 것으로, 정보 단말 장치를 추가하기 용이한 구성이다.
  
  장점 : 네트워크 관리가 쉽고 확장이 편리하며 네트워크에 대한 신뢰도가 높다.
  
  단점 : 특정 노드에 트래픽이 집중되면 네트워크 속도가 떨어질 수 있으며, 병목 현상이 발생할 수 있다.
  
  > 병목 현상 : 전체 시스템의 성능이나 용량이 하나의 구성 요소로 인해 제한받는 현상

- **Star Topology(성형 토폴로지)**

  중앙 시스템을 반드시 거쳐서 통신하는 방식(= 중앙에 있는 정보 단말 장치에 모두가 연결된 구조로, 항상 중앙의 정보 단말 장치를 통해서만 서로 연결이 가능한 구조)
  
  장점
  
    - 고속의 네트워크에 적합하다.
    - 노드 추가가 쉽고 에러를 탐지하기 용이(관리, 제어가 용이)하다.
    - 노드에 장애가 발생해도 네트워크는 사용 가능하다.
  
  단점
  
    - 중앙의 노드에 장애가 발생하면 전체 네트워크를 사용할 수 없다(모든 통신이 불가능하다).
    - 설치 비용이 고가이고 노드가 증가할수록 네트워크 복잡도가 증가한다.

- **Bus Topology(버스형 토폴로지)**

  ![image](https://user-images.githubusercontent.com/37951612/79134880-6dba4f80-7de9-11ea-892c-3bf930e08013.png)

  네트워크상의 모든 호스트들이 하나의 케이블로 연결되어 있는 방식(= 중앙의 통신 회선 하나에 여러 개의 정보 단말 장치가 연결된 구조), LAN에서 주로 사용된다.
  
  장점
  
    - 한 클라이언트가 죽어도 네트워크에는 지장이 없다.
    - 설치비용이 적고 신뢰성이 우수하다.
    - 구조가 간단하다.
    - 네트워크에 노드 추가가 쉽다(= 확장에 용이하다).
    
  단점
  
    - 전송되는 데이터가 많으면 네트워크 병목 현상이 발생한다.
    - 장애 발생 시 전체 네트워크에 영향을 받는다.
  
  > Terminator(터미네이터) : 목적지 없는 데이터를 잡는 역할을 하며, 시그널의 반사를 방지하기 위해 사용된다.

- **Ring Topology(링형 토폴로지)**

  양 종단이 서로 연결되어 링 형태를 이룬 토폴로지(= 인접해 있는 정보 단말 장치가 연결된 구조)
  
  ex) Token Ring(토큰 링)
  
  토큰이 존재하여 토큰을 가진 클라이언트만 통신이 가능하며, 토큰은 링을 돌며 통신 권한이 있는 클라이언트를 찾는다.
  
  장점
  
    - 노드의 수가 증가해도 데이터 손실이 없다.
    - 토큰을 가진 클라이언트만 통신을 하기 때문에 충돌이 없다.
    - 경제적인 네트워크 구성이 가능하다.
    
  단점
  
    - 토큰이 없는 클라이언트는 통신하기 위해 자신의 차례를 기다려야하기 때문에 속도 및 효율성이 떨어진다.
    - 네트워크 구성의 변경이 어렵다.
    - 회선에 장애 발생 시 전체 네트워크를 사용할 수 없다.

- **Mesh Topology(망형 토폴로지)**

  모든 네트워크 또는 컴퓨터들이 네트워크 상이나 개별적으로 연결된 방식(= 모든 정보 단말 장치가 통신회선을 통해서 연결된 구조로, 한쪽 통신회선에 에러가 발생해도 통신을 수행할 수 있는 구조다)
  
  장점
    
    - 완벽하게 이중화가 되어있으므로 장애 발생 시 다른 경로를 통해서 네트워크를 사용할 수 있다.
    
    - 많은 양의 데이터를 송수신할 수 있다.
  
  
  단점 : 네트워크 구축 비용, 운용 비용이 고가이다.
  
  
### 회선 교환 (Circuit Switching)
ex) 전화할 때 A, B라는 사람이 있다. A가 B에게 전화를 걸면 B 휴대폰은 전화를 받을 때까지 울리고, 만약 B가 다른 사람과 통화 중이면 A는 B가 통화 중이라는 사실을 알고 대기하게 되며, B가 전화를 받으면 그때부터 안정적으로 A와 통화가 가능하다. 즉, 전화는 A와 B간의 회선을 독점하는 것이다. -> **회선교환**

회선 교환은 한 번 연결이 이루어지면 안정적으로 통신할 수 있다. 또한 연결이 이루어지면(= 회선을 독점해서 사용) 자원을 많이 사용하고 다중 통신이 어렵다는 문제가 있다.

회선 교환은 Point to Point 방식으로 연결을 확립하고 안정적으로 통신할 수 있는 방법이다.

> QoS(Quality of Service) : 네트워크 품질을 평가하는 지표를 의미하며, QoS가 가장 우수한 네트워크 -> 회선 교환이다.

#### 장점
- 대용량의 데이터를 고속으로 전송하는 경우 좋다.
- 고정적인 대역폭(Bandwidth)을 사용한다.
- 접속에는 긴 시간이 소요되나 접속 이후에는 접속이 항상 유지되어 전송 지연이 없으며, 데이터 전송률이 일정하다.
- 아날로그나 디지털 데이터로 직접 전달한다.
- 연속적인 전송에 적합하다.

#### 단점
- 회선 이용률 측면에서 비효율적이다.
- 연결된 두 장치는 반드시 같은 전송률과 같은 기종 사이에 송수신이 요구된다(-> 다양한 개체 간 통신 제약).
- 속도나 코드의 변환이 불가능하다(교환망 내에서의 에러 제어 기능이 어렵다).
- 실시간 전송보다 에러 없는 데이터 전송이 요구되는 구조에서는 부적합하다.
- 통신 비용이 고가이다.


### 패킷 교환 (Packet Switching)

#### 패킷(Packet)

- 네트워크를 사용하여 전송하기 위해 데이터를 일정한 단위로 나눈 **전송 단위**
- 데이터 송신자와 수신자가 하나의 단위로 처리하는 **데이터 처리 단위**

#### 패킷 교환 개념

패킷 교환은 송신 측에서 모든 메시지를 일정한 크기의 패킷을 분해해서 전송하고, 이를 수신 측에서 원래의 메시지로 조립하는 것이다.

IP(Internet Protocol) 주소를 할당하여 네트워크에 데이터를 보내면 IP 주소를 확인한 후에 데이터를 전송한다.

인터넷은 회선교환과 다르게, 경로를 독점적이나 고정적으로 사용하지 않고 네트워크의 상태(ex. 속도, 대역폭)에 따라 다른 경로로 발송한다(=> 라우터. 네비게이션처럼 데이터를 전송하기 위해 최적의 경로를 선택하는 장비).

패킷 교환은 전송할 패킷에 대해 우선순위가 같은 것을 표시해서 중요한 패킷을 식별할 수 있도록 한다. 패킷 교환 네트워크는 공중 교환 데이터망(Public Switched Data Network)에서 사용된다.

#### 패킷 교환의 특징
- **다중화**: 패킷을 여러 경로로 공유한다.
- **채널**: 가상 회선 혹은 데이터그램 교환 채널을 사용한다.
- **경로 선택**: 패킷마다 최적의 경로를 설정한다.
- **순서 제어**: 패킷마다 최적의 경로로 보내기때문에 도착 순서가 다를 수 있다. 패킷의 순서를 정하여 도착 순서가 다르더라도 데이터가 순차적으로 읽히도록 한다.
- **트래픽 제어**: 전송 속도 및 흐름을 제어한다.
- **에러 제어**: 에러를 탐지하고 재전송한다.

#### 패킷 교환 장점
- 회선 이용률이 높고, 속도 변환, 프로토콜 변환이 가능하며, 음성 통신도 가능하다.
- 신뢰성이 높다.

  (경로 선택, 전송 여부 판별 및 장애 유무) 상황에 따라 교환기 및 회선에 장애가 발생하더라도 패킷의 우회 전송이 가능하다.
  
- 고품질

  디지털 전송이므로 인접 교환기 간에, 또는 단말기와 교환기 간에 전송 오류검사를 실시하여 오류 발생 시 재전송이 가능하다.

- 고효율

  다중화를 사용하므로 사용 효율이 좋다.

- 이기종 단말장치 간 통신

  전송 속도, 전송 제어 절차가 다르더라도 교환망이 변환 처리를 제공하므로 이기종 단말장치 간에 통신이 가능하다.

#### 패킷 교환 단점
- 경로 중 각 교환기에서 다소 지연이 발생한다. 이 지연은 가변적이며, 전송량이 증가함에 따라 지연이 더욱 심해질 수도 잇다.
- 패킷별 헤더 추가로 인해 오버헤드 발생 가능성이 존재한다.


### 데이터그램과 가상회선 (Datagram & Virtual Circuit)

> 데이터그램 네트워크는 패킷교환 방식으로 동작하면서 IP 주소를 사용하는 인터넷을 의미한다.

#### 가상회선(Virtual Circuit)

> ATM(Asynchronous Transfer Mode): 회선 교환의 실시간성 및 패킷 교환의 유연성을 통합시킨 연결지향적 패킷교환 복합기술 (대표적인 가상회선 방식의 통신기술). 특정 영역에서 사용하며 대중적으로 사용하는 데이터그램 네트워크보다는 공유 면에서 한계가 존재한다.

가상회선은 회선교환 방식과 데이터그램 방식의 장점을 결합한 통신 기술이다. **첫 패킷으로 최적의 경로를 고정하고 고정된 경로로 패킷을 고속으로 전송하는 기술**이며, 데이터그램보다 **더 빠르고 안정적으로 통신**할 수 있지만 데이터그램 네트워크처럼 **많은 사용자가 동시에 사용하기에는 한계**가 존재한다.

#### 데이터그램(Datagram) 특징
각 전송 패킷을 미리 정해진 경로 없이 독립적으로 처리하여 교환하는 방식이다.

같은 목적지의 패킷도 같은 경로를 거치지 않고, 서로 다른 경로를 통해 목적지에 도달한다. 따라서 망의 일부분이 혼잡할 때 전송 패킷에 다른 경로를 배정할 수 있으므로 융통성 있게 경로가 설정된다.

특정 교환기의 고장 시 모든 패킷을 잃어버리는 가상회선과 달리, 그 경로를 피해서 전송할 수 있으므로 신뢰성이 높아진다.

짧은 메시지의 패킷들을 전송할 때 효과적이고, 재정렬 기능이 필요하다.

#### 가상회선(Virtual Circuit) 특징
패킷을 전송하기 전, 논리적인 연결을 먼저 수행한다(-> 제어 패킷에 의한 연결형 서비스 제공). 송신자가 호출을 하고 호출 수신 패킷을 주고받아서 연결하는 방식이다. 

회선 교환처럼 사용하지만, 교환기에 패킷이 일시적으로 저장되기 때문에 일정한 전송률을 보장할 수 없다.

긴 메시지의 전송 시 더 효과적이다.

이미 확립된 접속을 끝내기 위해 Clear Request 패킷을 이용한다.


### 메시지 교환 (Message Switching)
메시지 교환은 송신된 메시지를 중앙에 축적하여 처리하는 방법으로, 흔히 **축적 교환 방식**이라고 한다. 메시지를 메모리에 저장하고 여러 수신자에게 데이터를 전송할 수 있으며, 전자우편에서 사용된다.

> 축적 교환 방식: 송신자가 메시지를 전송하면 메시지를 일정한 단위로 나누어서 버퍼에 저장하고, 저장이 완료되면 다시 그것을 읽어들여서 전송 경로를 결정하는 전송 방식이다. 인터넷도 축적 교환 방식으로 메시지를 전송한다.

- 메시지를 공유하여 데이터를 보낼 수 있다.
- 메시지에 우선순위를 부여한다.
- 에러 제어를 제공한다.
- 응답 속도가 느리다.
- 대화형 시스템으로 사용하기 어렵다.

### 근거리 통신 (LAN, Local Area Network)

제한된 지역 내에 설치된 각종 정보기기들 사이에서 통신을 수행하기 위해 구성된 고속의 통신 채널을 제공하는 것이 **근거리 통신망**이다. 일반적으로 50m 정도의 전송 거리를 가지고있다. 비교적 작은 범위의 네트워크망이다.

ex) **Ethernet, Token Ring, FDDI** 형태로 구축 가능

#### LAN 특징
- 10Mbps ~ 100Mbps 속도로 데이터를 전송한다.
- 멀티미디어 데이터를 전송할 수 있다.

> 채널(Channel): 통신 매체에서 제공하는 통로로 채널을 점유하여 통신이 이루어지게 된다. ex) TV 채널 점유하여 청취

- LAN의 목적: 자원 공유, 분산 처리, 분산 제어, 정보 교환
  - 자원 공유
  
    원격지의 자원을 공유하고, 복수의 사용자에게 독점적 사용권을 부여한다.
  
  - 분산 처리
  
    독립된 각 장비에서 계산과 작업을 처리하며, 전체 시스템의 능력은 연결된 모든 컴퓨터의 능력에 따라 결정된다.
  
  - 분산 제어
  
    분산된 지역의 독립적인 장치 간의 통신을 통해 프로세스를 제어하며 높은 데이터 전송 속도, 신뢰도를 유지한다.
  
  - 정보 교환
  
    Video & Voice 신호 전달, Text 데이터 전달

#### LAN 통신 장점
- 전송되는 패킷 손실 및 지연이 적다.
- 사용자 간에 쉽고 빠른 데이터 공유가 가능하다.
- 신뢰성이 높고 구축 비용이 적다.
- 오류 발생률이 낮다.

#### LAN 통신 단점
- 전송 거리에 제약이 있다(짧다).
- UTP 및 광케이블로 구축되지만, 네트워크에 노드가 많아지면 충돌이 발생하여 성능이 떨어진다.


### 이더넷 (Ethernet)
네트워킹의 한 방식으로, 이더넷의 가장 큰 특징은 **CSMA/CD** 프로토콜(LAN 통신 프로토콜)을 이용하여 통신한다는 점이다. 대부분의 인터넷 방식이 이더넷 방식이다. 

이더넷은 LAN(Local Area Network)를 위해 개발된 근거리 유선 네트워크 통신망 기술이다. 일반적으로 동축 케이블 또는 비 차폐 연선을 사용하고, 버스 형식으로 망을 구성한다.

#### 이더넷 장점
- 적은 용량의 데이터를 전송할 경우 성능이 우수하다.
- 설치 비용이 저렴하고 관리가 쉽다.
- 네트워크 구조가 단순하다

#### 이더넷 단점
- 네트워크 사용 시 신호 때문에 충돌이 발생하고, 충돌이 발생하면 네트워크 지연이 발생한다.
- 시스템 부하가 증가하면 충돌도 계속적으로 증가한다.

#### 이더넷 표준
- `10Base-5` : Thick 케이블이라고도 부르며 500m 길이를 가지는 동축 테이블이다. 2.5m 간격으로 트랜시버를 연결하여 사용한다.
- `10Base-2` : Thin 케이블이라고도 부르며 200m의 길이를 가진다.
- `10Base-T` : UTP 케이블을 이용하는 것으로 현재 많이 사용 된다. (100m의 길이)

#### 기가비트 이더넷
1초에 1Gbps 속도로 데이터를 전송할 수 있는 이더넷 표준 기술이다. 호환성이 좋아서 기존 이더넷과 호환되며, `100Base-X`로 정의되었다. CSMA/CD 프로토콜과 10Mbps~100Mbps의 기존 이더넷 형태를 유지하며 이더넷과 고속 이더넷의 관리 시스템 이용이 가능하다.

Star Topology를 사용하며, MAC 프로토콜로 CSMA/CD 방식을 사용한다. 고속 이더넷보다 고가지만, 10배의 전송 속도를 가진다. 케이블의 길이를 줄이고 전송 속도를 향상한 것이다.


### CSMA(Carrier Sense Multiple Access) / CD(Collision Detection) 프로토콜
LAN의 통신 프로토콜 종류 중 하나로, 이더넷 환경에서 사용한다. `대충 알아서 눈치껏 통신하자`는 방식의 프로토콜이다.

1. 이더넷 환경에서 통신하고싶은 PC나 서버는 먼저 네트워크 상에 통신이 일어나는지 확인한다(-> Carrier Sense).

  > Carrier: 네트워크 상에 나타나는 신호. 캐리어가 감지되면(네트워크 통신이 일어나고 있다면) 데이터를 보내지 않고 기다린다.
  
2. 만약 캐리어가 감지되지 않았을 때 두 PC나 서버가 데이터를 동시에 보내면 이 경우를 Multiple Access(다중 접근)라고 한다.
3. 두 PC나 서버가 데이터를 동시에 보내려다 부딪치는 경우 충돌(Collision)이 발생했다고 한다. 만약 충돌이 발생했다면 데이터를 전송했던 두 PC나 서버들은 랜덤한 시간동안 기다린 다음 다시 데이터를 전송하고, 이런 충돌이 15번 일어나면 통신을 끊는다.

#### CSMA/CD 충돌하지 않을 경우
![image](https://user-images.githubusercontent.com/37951612/79157198-25ae2380-7e0f-11ea-8580-fc91d9b5d4e9.png)

1. 전송을 원하는 호스트는 네트워크에 캐리어를 감지해서 전송이 가능한지 검사한다. ex) A에서 D로 데이터프레임 전송
2. 전송이 가능한 경우, A에서 B,C,D로 브로드캐스트 하여 D를 찾아낸다(-> D는 유니캐스트로 응답한다).
3. 전송한다.

#### CSMA/CD 충돌할 경우
![image](https://user-images.githubusercontent.com/37951612/79157204-28a91400-7e0f-11ea-87bf-e2f09c222fcd.png)

1. 전송을 원하는 호스트는 네트워크에 캐리어를 감지해서 전송이 가능한지 검사한다. ex) A에서 D로, B에서 C로 데이터프레임 전송
2. A에서 발생한 프레임, B에서 발생한 프레임은 공유 매체에서 충돌이 발생한다.
3. 충돌이 발생하면 Jam Signal을 모든 호스트에 전송하여 충돌을 알리고, Jam Signal을 받으면 일정시간 뒤 다시 전송을 시도한다.


### 토큰패싱 (Token Passing)

토큰이라는 제어 비트를 송신하고, 해당 토큰을 확보하여 통신하는 방식이다. 통신 회선에 대한 제어 신호가 논리적으로 형성된 링(-> Ring Topology) 내의 각 노드를 옮겨가면서 데이터를 전송하는 방식이다. 토큰을 가지고 있는 노드만 통신이 가능하므로 충돌이 발생하지 않는다.

#### 특징
- 가변 길이의 데이터 프레임 전송이 가능하다.
- 하드웨어 장비가 복잡하고 평균 대기 시간이 높다.
- 부하가 높을 때 안정감이 있고, 접근 시간이 일정한 값을 유지한다.
- Ring Topology 통신망에서 제어 신호가 각 노드 간을 순차적으로 옮겨가면서 데이터를 전송한다.
- 특정한 비트 패턴으로 구성된 짧은 프레임 형태이다.
- 통신 회선의 길이가 무제한이다.
- 확장이 어렵다.
- 고속 버스트 전송에 유지하다.

#### 장점
- 충돌이 발생하지 않는다.
- 성능 저하가 적다.

#### 단점
- 설치 비용이 고가이다. 복잡하다.
- 노드가 많을수록 성능이 떨어진다.
- 토큰 분실이 발생할 수 있다.


### 정보신호 (Information Signal)

**아날로그 신호와 디지털 신호**로 나눌 수 있다.

- 아날로그 신호

  연속적으로 변화하는 전자기파로, 곡선형 형태로 나타난다. 거리가 멀어지면 점점 감쇄되는 현상이 발생한다.

- 디지털 신호

  데이터를 0과 1로 표현하며, 아날로그 신호에 비해 잡음이 적고 오류율이 적다.

- 아날로그 -> 아날로그

  **증폭기**를 이용하여 신호의 세기를 증폭한다. 잡음까지 증폭되고, 왜곡이 심하다.
  
  ex) 전화기

- 아날로그 -> 디지털

  **코덱**을 이용한다. 원음만을 재생하여 왜곡 현상을 방지한다.
  
  ex) PCM
  
  > PCM(Plus Code Modulation): 컴퓨터로 목소리를 녹음할 때, 아날로그 신호(목소리)를 디지털 신호로 변환하는 것을 의미한다.
  
- 디지털 -> 아날로그

  **모뎀**을 이용한다. 아날로그 통신망을 이용하여 디지털 신호를 전송한다.
  
- 디지털 -> 디지털

  **DSU**를 이용한다. 적당한 간격으로 리피터를 설치한다.
  
  > DSU(Digital Service Unit): 디지털 신호를 디지털로 전송할 때, 신호를 안정적으로 멀리 보내기 위해 변환하는 것이다.
  
아날로그 신호의 단점은 **유지보수 비용 증가, 잡음 증폭도 높음**이다.

디지털 신호의 장점은 **저렴한 비용, 데이터 무결성 보장, 전송 용량 확대, 데이터 안정성 증대(암호화 작업), 정보 종합**이다.
